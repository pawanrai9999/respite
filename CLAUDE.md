## Project

Respite is a break-reminder application for the GNOME desktop, written in **C** using **GTK4 + libadwaita**, targeting **Wayland**. The user sets a work interval and a break length; when the interval elapses Respite blacks out every monitor with a centered `Break for MM:SS` countdown, allows a limited number of postponements, and is meant to run as an autostarting background daemon.

**Current state:** Phases 1 (settings foundation), 2 (timer engine), 3 (process model & daemon), 4 (break overlay), 5 (postpone mechanism), 6 (autostart & background), and 7 (polish & release readiness) are complete. Phase 7 holds a `gtk_application_inhibit()` session inhibitor (suspend + idle) across each break, finalizes the About dialog / desktop entry / AppStream metainfo, completes `po/POTFILES.in` and regenerates `po/respite.pot`, removes the `--debug-timer` scaffolding from `main.c`, and fixes the Flatpak manifest (runtime 50, corrected git source) to build; the optional idle-handling item (7.1) was skipped as unreliable under sandboxed GNOME. The "Hello World" template has been replaced with an Adwaita preferences window whose controls are bound two-way to GSettings with input constraints enforced; the window also carries a live **Status** group (`src/respite-window.c` borrows the shared timer once `notify::application` fires and mirrors phase/countdown/postpones-remaining on every `tick`/`state-changed`) and a Pause/Resume button wired to the `app.toggle-timer` action. `RespiteApplication` drives this through `respite_application_set_active()`, which keeps the daemon `g_application_hold` across a pause (so a paused process stays alive and ready to resume) and releases it only on quit. The UI builds on a UI-agnostic `RespiteTimer` GObject (`src/respite-timer.c`) drives the work/break countdown via a monotonic clock, emitting `tick`/`state-changed`/`break-started`/`break-ended` signals. `RespiteApplication` owns a single shared `RespiteTimer` created in `startup`; the app uses `G_APPLICATION_HANDLES_COMMAND_LINE` so a `--daemon` invocation runs headless (takes a `g_application_hold`, starts the timer, no window) while any other invocation presents/raises the single settings window, and the quit/shutdown path stops the timer and releases the hold. On `break-started` the application covers every monitor with a `RespiteOverlay` (`src/respite-overlay.c`) — a fullscreen, undecorated black `GtkWindow` showing a centered `Break for MM:SS` countdown styled from `respite-overlay.css`; `tick` updates the count, `break-ended` tears the overlays down, and monitor hotplug is reconciled live during a break. The overlay is a best-effort nudge (grabs focus, refuses close, re-presents) — not an unbreakable lock, since Mutter does not honor `wlr-layer-shell`. The timer also tracks a per-cycle postpone allowance (seeded from `postpone-allowance`, reset each work cycle): at `pre-break-warning` seconds before a break it moves WORKING → WARNING and emits `warning`, on which the application raises a notification with a Postpone button wired to the `app.postpone` action; `respite_timer_postpone()` pushes the break back by `postpone-duration` and returns to WORKING. Background and autostart live in `src/respite-background.c`: in daemon mode the app asks the XDG Background portal (`RequestBackground` over raw GDBus) for permission to run windowless, and the `Start on Login` toggle drives `respite_background_set_autostart()`, which registers/removes login launch of `respite --daemon` through the same portal when sandboxed or via an `~/.config/autostart/io.github.pawanrai9999.respite.desktop` entry otherwise; `RespiteApplication` watches `changed::autostart` and writes the portal's actual verdict back into GSettings (see `PLAN.md` for the phased roadmap and current status). The README's "Behavior in Detail" and "Roadmap" sections describe the intended design.

**Strict break enforcement (0.3.0, optional).** Because Mutter does not let a client grab global input, the GTK overlay above is only a best-effort nudge; for hard enforcement Respite ships a companion **GNOME Shell extension** (in `extension/`, UUID `respite-strict-break@pawanrai9999.github.io`) that runs inside the compositor and takes a `Main.pushModal` modal grab (`actionMode: NONE`, which filters every keybinding) over a black full-screen actor, so Alt+Tab/Super/the overview are all suppressed. The daemon stays the brain; the extension is a dumb actuator driven over the session bus per the shared contract `extension/dbus-interface.xml` (interface `io.github.pawanrai9999.respite.Strict`, object `/io/github/pawanrai9999/respite/Strict`): methods `StartBreak(u)→b` / `UpdateRemaining(u)` / `EndBreak()` / `Ping()→s` and signals `Released` / `EscapeRequested`. The extension self-protects so a grab can never trap the session — the grab handle is stored the instant `pushModal` returns (wrapped in try/catch), a heartbeat watchdog auto-releases if the daemon stops driving it (~10s), and holding `Esc` ~3s emits `EscapeRequested`. Its `captured-event` handler stops only keyboard events and lets the rest propagate (stopping crossing/motion events trips a Clutter `EVENT_PROPAGATE` check). On the daemon side, `RespiteStrict` (`src/respite-strict.c`) is a GObject D-Bus client mirroring `respite-background.c`'s raw-GDBus style: it tracks the extension's bus name with `g_bus_watch_name` (emitting `availability-changed`), subscribes to `Released`/`EscapeRequested`, and issues fire-and-forget calls that no-op when the extension is absent. `RespiteApplication` owns it (`respite_application_get_strict`); when the `strict-break` GSetting is on and the extension is available, `on_break_started` drives the lockdown instead of `show_overlays`, `tick` heartbeats `UpdateRemaining`, and break teardown is unified through `respite_application_end_active_break()` (run on `break-ended`, on pause, and at shutdown so a paused/quit strict break releases the grab promptly). A failed or dropped grab arrives as `::released` and falls the daemon back to the GTK overlay for the rest of the break; the escape hatch arrives as `::escape-requested` and ends the break early via `respite_timer_end_break()`. The prefs window's **Strict Break** toggle is gated insensitive (with an inline install-prompt row) until the extension owns its name — `src/respite-window.c` borrows `RespiteStrict` like it borrows the timer and mirrors `availability-changed`. The extension is a **separate artifact** with its own lifecycle and version (`extension/metadata.json`): it cannot live in the Flatpak (a sandboxed app cannot install Shell extensions), so it ships via the repo `extension/` (use `extension/install.sh` to install/pack; `extension/test-dbus.sh` to exercise it) and, eventually, extensions.gnome.org. **Verify Shell/Clutter APIs against the installed Shell before using them** — `gresource extract /usr/lib/gnome-shell/libshell-18.so /org/gnome/shell/ui/<file>.js` for JS, and the `mutter-18` typelib for Clutter enums — rather than trusting docs/plans.

## Build & Run

The GNOME Builder IDE and the build toolchain (meson, ninja, gtk4/libadwaita dev files) are **not installed natively on the host** — they are provided through Flatpak: GNOME Builder is installed as a Flatpak app, and meson/ninja/etc. come from the `org.gnome.Sdk` runtime (versions **49 and 50** are installed; the manifest pins 50). To compile against that toolchain without Builder, run meson inside the SDK runtime:

```sh
flatpak run --filesystem=/home/pawan/Projects/Respite --command=sh org.gnome.Sdk//50 -c \
  'meson setup _build && meson compile -C _build'
```

Standard Meson workflow (used inside Builder or any environment where the toolchain is on `PATH`):

```sh
meson setup _build
meson compile -C _build
./_build/src/respite          # run the built binary
meson install -C _build       # optional, installs system-wide
```

Flatpak build (manifest `io.github.pawanrai9999.respite.json`, GNOME Platform/SDK 50):

```sh
flatpak-builder --user --install _flatpak io.github.pawanrai9999.respite.json
```

Note: the manifest's git source points at the GitHub remote pinned to the latest released tag/commit (currently `0.3.0`), so a Flatpak build pulls that published commit rather than the working tree. To build the local checkout instead, override the source to `{"type": "dir", "path": "."}` (as the CI workflow does). The Pages release workflow (`.github/workflows/pages.yml`) builds the manifest **as-is**, so it ships whatever commit the pin names — cutting a release means bumping the version (`meson.build` + a metainfo `<release>` entry), tagging it, and then a follow-up commit that re-pins the manifest to that tag/commit (the pattern of the "Pin the X.Y.Z release commit" commits).

There is no test suite or linter configured. Build warnings are extensive (see the `test_c_args` list in the top-level `meson.build`) but `werror=false`, so warnings do not fail the build.

## Architecture & Conventions

- **App ID / namespace:** everything uses `io.github.pawanrai9999.respite`. GResource base path is `/io/github/pawanrai9999/respite`; C symbols use the `respite_` prefix and `RESPITE_` / `Respite` type macros.
- **Object model:** GObject-based. `RespiteApplication` (subclasses `AdwApplication`) owns app-level actions (`app.quit`, `app.about`) and creates `RespiteWindow` (subclasses `AdwApplicationWindow`) on `activate`. Entry point is `src/main.c`.
- **UI is defined in `.ui` templates, not C.** `src/respite-window.ui` and `src/shortcuts-dialog.ui` are GTK Builder XML, bundled via `src/respite.gresource.xml` and loaded with `gtk_widget_class_set_template_from_resource`. Widgets are wired into the struct with `gtk_widget_class_bind_template_child`. To add UI, edit the `.ui` file, register the resource in the gresource XML, and bind template children in the corresponding `*_class_init`.
- **Three meson layers:** root `meson.build` sets up i18n/gnome modules, generates `config.h`, and recurses into `src/`, `data/`, and `po/`. Source files and dependencies are declared in `src/meson.build`.
- **Settings:** a GSettings schema exists at `data/io.github.pawanrai9999.respite.gschema.xml` — this is where persisted preferences (interval, break time, postpone allowance, autostart, `strict-break`) belong.
- **Companion extension:** the strict-break GNOME Shell extension lives in `extension/` (ESM, targets shell 45–50). It is a separate artifact — not built by meson, not in the Flatpak, with its own `metadata.json` version. It is coupled to the C daemon only through the D-Bus contract `extension/dbus-interface.xml`; edit that one file to change the interface and keep both sides in sync.
- **Data/integration files** live in `data/`: `.desktop`, D-Bus `.service`, AppStream `.metainfo.xml`, icons under `data/icons/hicolor/`.
- **i18n:** gettext package is `respite`. New translatable source files must be added to `po/POTFILES.in`; wrap user-facing strings in `_()`.
- **License:** AGPL-3.0-or-later. New source files should carry the same SPDX header block as existing files in `src/`.
